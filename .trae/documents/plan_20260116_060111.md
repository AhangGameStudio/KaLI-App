# KaLI-App 开发计划

## 项目现状分析

* 现有项目：KaLI-App，包含PC端(Tkinter)和移动端(Kivy)界面

* 已集成Wifite2无线攻击工具

* 具备Android打包构建配置

* 当前功能：Kali Linux工具启动器

## 开发目标实现计划

### 1. 自检程序（PC端）

**功能模块**：

* 硬件配置检测：CPU、内存、磁盘、显卡信息

* 系统状态监控：CPU/内存使用率、磁盘空间、网络连接

* 性能指标分析：启动时间、响应速度、资源占用

* 潜在问题诊断：驱动更新、系统漏洞、硬件异常、一键优化、

**技术实现**：

* 使用`psutil`库获取系统信息

* 使用`platform`库获取硬件信息

* 扩展现有Tkinter界面，添加新的"系统自检"标签页

* 实现实时数据采集和历史数据图表展示

### 2. 网络安全程序（PC/移动端跨平台）

**功能模块**：

* 实时网络流量监控：数据包捕获与分析

* 恶意软件检测：基于特征和行为分析

* 入侵防御：异常连接检测与阻止

* 安全漏洞扫描：端口扫描、服务版本检测

**技术实现**：

* PC端：使用`scapy`库进行网络数据包分析

* 移动端：使用Kivy接口，调用系统网络API

* 跨平台代码结构：核心逻辑共享，UI层分离

* 实现本地安全规则库和在线更新机制

### 3. 无线WIFI扫描与破解工具（移动端）

**功能模块**：

* WIFI网络扫描：SSID、信号强度、加密类型

* 网络信息获取：BSSID、信道、MAC地址

* 密码破解：支持WEP、WPA/WPA2常见破解方法

* 连接管理：已破解网络保存与自动连接

**技术实现**：

* 基于现有Wifite2工具进行移动端适配

* 使用Kivy构建移动界面

* 集成`pywifi`库进行WIFI操作

* 实现字典攻击和PIN码破解功能

### 4. 漏洞挖掘工具（移动端）

**功能模块**：

* 应用程序静态分析：权限清单、代码审查

* 动态行为监控：网络请求、文件操作、权限使用

* 敏感权限检测：危险权限使用分析

* 潜在安全漏洞识别：常见漏洞模式匹配

**技术实现**：

* 使用`androguard`库进行APK静态分析

* 实现运行时行为监控模块

* 构建漏洞特征数据库

* 生成详细的安全评估报告

## 项目结构调整

```
KaLI-App/
├── core/              # 核心功能模块（跨平台共享）
│   ├── system/        # 系统自检相关
│   ├── network/       # 网络安全相关
│   ├── wireless/      # WIFI工具相关
│   └── security/      # 漏洞挖掘相关
├── pc/                # PC端特有代码
│   ├── ui/            # Tkinter界面
│   └── main.py        # PC端入口
├── mobile/            # 移动端特有代码
│   ├── ui/            # Kivy界面
│   └── mobile_app.py  # 移动端入口
├── tools/             # 第三方工具
│   └── wifite2/       # 无线攻击工具
├── data/              # 数据文件
│   ├── signatures/    # 安全特征库
│   └── wordlists/     # 密码字典
├── tests/             # 测试代码
├── docs/              # 文档
└── build/             # 构建配置
    ├── .recipes/      # 构建配方
    ├── build.yml      # GitHub Actions配置
    └── buildozer.spec # Buildozer配置
```

## 依赖管理

* **核心依赖**：

  * `psutil`：系统信息获取

  * `scapy`：网络数据包分析

  * `pywifi`：WIFI操作

  * `androguard`：APK分析

  * `matplotlib`：数据可视化

  * `requests`：网络请求

* **平台特定依赖**：

  * PC端：`tkinter`（标准库）

  * 移动端：`kivy`、`pyjnius`

## 构建与部署

* 保持现有Android构建配置

* 每完成一个任务/一大步上传一次

* 为PC端添加打包配置（Windows .exe、Linux .deb）
